# Module 3 — Shared Data Models

## Purpose

Modules 0–2 built the project scaffold and the LLM communication
layer. Before any agent, planner, supervisor, or workflow is
introduced, the project needs a shared, strongly typed vocabulary for
the things those future components will pass to each other: tasks,
messages, and agent identities.

This module introduces `app/schemas`, the package defining those
shared contracts. **Every future module must communicate using these
models — never raw dictionaries or bare strings.** This module
contains no AI logic, no persistence, and no business behavior; it
defines data shapes only.

## Responsibilities

This module is responsible for:

- Defining `Message`, the typed contract for communication between
  two parties (e.g. future agents).
- Defining `Task`, the typed contract for a unit of work.
- Defining `AgentInfo`, the typed contract describing an agent's
  identity and role.
- Defining the enums (`TaskStatus`, `TaskPriority`, `MessageType`,
  `AgentRole`) that constrain the above models' fields to a fixed,
  well-known vocabulary.

This module is explicitly **not** responsible for (and does not
implement): agents, LangGraph, prompts, memory, HTTP, tools,
databases, or any business logic that acts on these models. It only
defines what the data looks like.

## Architecture

```
app/schemas/
├── __init__.py     # Public exports
├── enums.py          # TaskStatus, TaskPriority, MessageType, AgentRole
├── message.py          # Message
├── task.py               # Task
└── agent.py                 # AgentInfo
```

These models have no dependency on `app.llm`, `app.core`, or any other
application package — they are pure data contracts, which keeps them
safely importable from anywhere in the codebase (including future
`app/agents`, `app/orchestration`, and `app/api` packages) without
risk of circular imports.

## Public API

### Enums (`app/schemas/enums.py`)

All four enums subclass both `str` and `Enum`, so they serialize to
plain strings (e.g. via `model_dump(mode="json")` or
`model_dump_json()`) while remaining fully usable as regular Python
enums.

| Enum | Members |
|---|---|
| `TaskStatus` | `PENDING`, `IN_PROGRESS`, `COMPLETED`, `FAILED`, `CANCELLED` |
| `TaskPriority` | `LOW`, `MEDIUM`, `HIGH`, `CRITICAL` |
| `MessageType` | `REQUEST`, `RESPONSE`, `NOTIFICATION`, `ERROR`, `SYSTEM` |
| `AgentRole` | `SUPERVISOR`, `PROJECT_MANAGER`, `RESEARCH_ANALYST`, `BUSINESS_ANALYST`, `TECHNICAL_ARCHITECT`, `FINANCIAL_ANALYST`, `SOFTWARE_ENGINEER`, `DOCUMENTATION_WRITER`, `REVIEWER`, `EVALUATOR` |

### `Message` (`app/schemas/message.py`)

| Field | Type | Default |
|---|---|---|
| `id` | `UUID` | auto-generated (`uuid4`) |
| `sender` | `str` | required, non-blank |
| `receiver` | `str` | required, non-blank |
| `content` | `str` | required, non-blank |
| `message_type` | `MessageType` | `REQUEST` |
| `timestamp` | `datetime` | auto-generated (UTC now) |

### `Task` (`app/schemas/task.py`)

| Field | Type | Default |
|---|---|---|
| `id` | `UUID` | auto-generated (`uuid4`) |
| `title` | `str` | required, non-blank |
| `description` | `str` | `""` |
| `priority` | `TaskPriority` | `MEDIUM` |
| `status` | `TaskStatus` | `PENDING` |
| `created_at` | `datetime` | auto-generated (UTC now) |
| `updated_at` | `datetime` | auto-generated (UTC now) |

`updated_at` is initialized automatically but is not touched
automatically on later mutation — updating it on change is the
responsibility of whichever future module owns task state transitions
(e.g. an orchestration layer), not this contract.

### `AgentInfo` (`app/schemas/agent.py`)

| Field | Type | Default |
|---|---|---|
| `id` | `UUID` | auto-generated (`uuid4`) |
| `name` | `str` | required, non-blank |
| `role` | `AgentRole` | required |
| `description` | `str` | `""` |

## Configuration

None. This module introduces no environment variables and depends on
no application settings.

## Usage Example

```python
from app.schemas import AgentInfo, AgentRole, Message, Task, TaskPriority

task = Task(
    title="Draft technical architecture",
    description="Propose a system design for the client's idea.",
    priority=TaskPriority.HIGH,
)

agent = AgentInfo(
    name="Ada",
    role=AgentRole.TECHNICAL_ARCHITECT,
    description="Designs technical architectures for client proposals.",
)

message = Message(
    sender=str(agent.id),
    receiver="supervisor",
    content=f"Started work on task {task.id}.",
)
```

## Acceptance Criteria

- [x] `Message`, `Task`, and `AgentInfo` are defined as strongly typed
      Pydantic models.
- [x] `TaskStatus`, `TaskPriority`, `MessageType`, and `AgentRole` are
      defined as Python enums.
- [x] IDs are UUIDs, auto-generated by default.
- [x] Timestamps are auto-generated by default.
- [x] Sensible defaults exist for optional fields
      (`priority`, `status`, `message_type`, `description`).
- [x] Invalid values (blank required strings, invalid enum values,
      malformed UUIDs) are rejected with a `pydantic.ValidationError`.
- [x] Models round-trip correctly through both `dict` and JSON
      serialization.
- [x] No AI logic, persistence, HTTP, or business behavior is present
      in this module.
- [x] Documentation (this file) and ADR-004 are complete.
- [x] `PROJECT_ROADMAP.md` is updated to mark Module 3 complete.
